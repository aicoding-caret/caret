# 작업 문서 작성 가이드 (계획, 체크리스트, 보고서)

## 1. 개요

이 문서는 Caret 프로젝트에서 사용되는 주요 작업 문서들(작업 계획, 작업 체크리스트, 작업 보고서)의 작성 방법과 표준 템플릿을 안내합니다. 일관되고 명확한 문서 작성은 원활한 작업 진행과 효율적인 커뮤니케이션의 핵심입니다.

**참고**: 일반적인 문서 작성 표준, 스타일, 용어 사용, 파일 명명 규칙 등 프로젝트 전반의 문서화 지침은 [Project Documentation Guide](../../development/documentation-guide.md)를 참조하세요.

모든 작업 문서는 `docs/tasks/` 디렉토리에 다음 명명 규칙을 따라 저장됩니다:
*   작업 계획: `<task-number>-01-plan-<task-name>.md`
*   작업 체크리스트: `<task-number>-02-action-checklist-<task-name>.md`
*   작업 보고서: `<task-number>-03-report-<task-name>.md`

새로운 작업 시작 전에는 `docs/tasks/task-status.md`를 확인하여 다음 작업 번호를 부여받아야 합니다.

## 2. 작업 계획 (`<task-number>-01-plan-*.md`)

### 2.1. 목적
작업 계획 문서는 특정 작업의 목표, 범위, 주요 산출물, 예상 기간, 필요한 사전 지식 및 참조 자료, 그리고 기본적인 행동 지침을 정의합니다. AI 어시스턴트가 작업을 시작하기 전 가장 먼저 숙지해야 하는 문서입니다.

### 2.2. 주요 섹션 및 작성 요령

*   **메타 정보**:
    *   `Task #<task-number>`: 작업 번호
    *   `제목`: 명확하고 간결한 작업 제목
    *   `작성일`: 문서 최초 작성일
    *   `작성자`: 문서 최초 작성자 (예: Luke Yang (개발자), Alpha Yang (AI 어시스턴트))
    *   `우선순위`: (예: High, Medium, Low)
    *   `예상 작업일/시간`: (예: 3일, 8시간)
    *   `목적`: 이 작업을 통해 달성하고자 하는 핵심 목표를 명확히 기술합니다.

*   **작업 전 숙지사항 (CRITICAL for AI Assistant)**:
    *   이 섹션은 AI 어시스턴트가 작업을 수행하기 위해 반드시 읽고 이해해야 하는 **핵심 업무 문서**들을 명시해야 합니다.
    *   문서 경로와 함께 해당 문서에서 어떤 정보를 얻어야 하는지 간략히 설명하는 것이 좋습니다.
    *   예시:
        ```markdown
        ### 핵심 업무 문서 (작업 방법 숙지용)
        *   `docs/development/caret-architecture-and-implementation-guide.md`: Caret의 오버레이 아키텍처 및 핵심 구현 원칙을 숙지합니다.
        *   `docs/development/logging.md`: 프로젝트 로깅 표준 및 `Logger`, `WebviewLogger` 사용법을 익힙니다.
        *   `.caretrules`: 프로젝트 전반의 개발 규칙 및 AI 행동 지침을 확인합니다.
        ```

*   **작업 배경 및 필요성 (선택 사항)**:
    *   해당 작업이 왜 필요한지, 어떤 문제를 해결하기 위한 것인지 간략히 설명합니다.

*   **주요 작업 목표 및 범위**:
    *   달성해야 할 주요 목표들을 구체적으로 나열합니다.
    *   "여기까지 포함하고, 저기는 포함하지 않음"과 같이 작업의 경계를 명확히 합니다.

*   **주요 산출물**:
    *   작업 완료 시 기대되는 결과물 목록 (예: 특정 기능 구현, 문서 업데이트, 테스트 케이스 작성 등).

*   **기본 행동 지침 (AI 어시스턴트 대상)**:
    *   AI 어시스턴트가 해당 작업을 수행할 때 특별히 유의해야 할 사항이나 따라야 할 원칙을 명시합니다. (예: 특정 파일은 수정 금지, 특정 라이브러리 우선 사용 등)

*   **참고 자료 (선택 사항)**:
    *   작업 수행에 도움이 될 수 있는 추가적인 문서, 웹사이트 링크 등을 제공합니다.

## 3. 작업 체크리스트 (`<task-number>-02-action-checklist-*.md`)

### 3.1. 목적
작업 계획에 명시된 목표를 달성하기 위한 구체적이고 실행 가능한 작업 항목들의 목록입니다. 각 항목은 완료 여부를 추적할 수 있어야 합니다.

### 3.2. 주요 섹션 및 작성 요령

*   **메타 정보**:
    *   `Task #<task-number>`: 작업 번호
    *   `제목`: "<작업 계획 제목> - Action Checklist"
    *   `관련 계획 문서`: 해당 작업 계획 문서로의 링크 (예: `[001-01-plan-caret-architect-initialize.md](001-01-plan-caret-architect-initialize.md)`)
    *   `목표`: "이 체크리스트는 `<관련 계획 문서>`에 명시된 작업 목표를 달성하기 위한 구체적인 실행 항목을 정의합니다."

*   **작업 항목 (`## ✅ 작업 항목`)**:
    *   주요 작업 목표별로 하위 섹션을 나누어 관리할 수 있습니다.
    *   각 항목은 명확한 동사로 시작하고, 구체적인 액션을 나타내야 합니다.
    *   Markdown의 체크박스(`- [ ]` 또는 `- [x]`)를 사용하여 완료 여부를 표시합니다.
    *   필요시 각 항목에 대한 간략한 설명이나 참조 링크를 추가할 수 있습니다.
    *   너무 크거나 모호한 항목은 여러 개의 작은 항목으로 분리합니다.
    *   예시:
        ```markdown
        ### 1. WelcomeView UI 구현
        - [ ] **1.1. 기본 레이아웃 구성**: `WelcomeView.tsx` 파일 생성 및 기본 JSX 구조 작성
        - [ ] **1.2. 스타일 적용**: `welcome-view.scss` 파일 생성 및 CSS 스타일링
        - [x] **1.3. 다국어 문자열 연동**: `useTranslation` hook 사용하여 UI 텍스트 적용 (참고: `docs/development/locale.md`)
        ```

*   **다음 단계 (선택 사항)**:
    *   체크리스트 진행에 대한 추가적인 안내나 다음으로 넘어갈 작업에 대한 언급을 할 수 있습니다.

## 4. 작업 보고서 (`<task-number>-03-report-*.md`)

### 4.1. 목적
작업 진행 상황, 발생했던 주요 이슈와 해결 과정, 그리고 최종 결과를 요약하여 기록하는 문서입니다. 이는 작업의 이력을 추적하고, 향후 유사 작업 시 참고 자료로 활용될 수 있습니다.

### 4.2. 주요 섹션 및 작성 요령

*   **메타 정보**:
    *   `Task #<task-number>`: 작업 번호
    *   `제목`: "<작업 계획 제목> - 작업 보고서"
    *   `작성일`: 보고서 작성 또는 최종 업데이트 날짜
    *   `관련 계획 문서`: 링크
    *   `관련 체크리스트`: 링크

*   **진행 상황 요약 (`## 1. 진행 상황 요약`)**:
    *   **완료된 작업**: 체크리스트 기준으로 완료된 주요 작업 항목들을 요약합니다.
    *   **현재 진행 중인 작업 (있다면)**: 아직 완료되지 않았지만 진행 중인 항목을 기술합니다.
    *   **다음 단계 (있다면)**: 현재 시점에서 계획된 다음 작업을 간략히 언급합니다.

*   **이슈 및 해결 방안 (`## 2. 이슈 및 해결 방안`)**:
    *   **현재 이슈 (있다면)**: 해결되지 않은 문제점, 원인 분석, 현재까지의 시도, 향후 해결 계획 등을 기술합니다.
    *   **해결된 주요 이슈**: 작업 중 발생했던 중요한 문제들과 이를 어떻게 해결했는지 상세히 기록합니다. (문제 상황, 원인, 해결 과정, 결과, 교훈 등)

*   **주요 결정 사항 (선택 사항)**:
    *   작업 진행 중 내려진 중요한 기술적/방법론적 결정 사항들을 기록합니다.

*   **최종 결과 및 산출물 요약 (`## 3. 최종 결과 및 산출물`)**:
    *   작업의 최종 결과를 요약하고, 주요 산출물(변경된 파일 목록, 새로 생성된 기능, 업데이트된 문서 등)을 명시합니다.
    *   필요시 스크린샷이나 코드 스니펫을 포함할 수 있습니다.

*   **결론 및 제언 (선택 사항)**:
    *   작업에 대한 전반적인 평가, 교훈, 향후 개선 사항 등을 제안합니다.

이 가이드라인을 따라 작업 문서를 작성하면 프로젝트의 모든 참여자가 작업 내용을 명확히 이해하고 효과적으로 협업하는 데 도움이 될 것입니다.
