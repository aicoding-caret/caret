# AI 업무 방법 가이드 (For Al Assistance)

## 1. 개요

이 문서는 AI 어시스턴트가 작업을 수행하는 표준 절차와 원칙을 정의합니다. AI는 이 가이드라인을 숙지하고 모든 작업을 이 절차에 따라 수행해야 합니다. 이 가이드의 목표는 AI가 독립적으로 작업의 맥락을 파악하고, 효율적이며 정확하게 업무를 처리할 수 있도록 지원하는 것입니다.

## 2. 작업 착수 프로토콜

개발자로부터 작업 지시를 받으면, AI는 다음 단계를 **반드시 순서대로** 수행해야 합니다. 코딩 또는 파일 수정을 시작하기 전에 이 프로토콜을 완료하는 것은 **매우 중요합니다 (CRITICAL)**.

1. **작업 로그 초기 검토**:
   * 오늘 날짜의 작업 로그 파일을 확인합니다 (`docs/work-logs/{username}/{date}.md`).
   * 작업 로그가 없으면 새로 생성합니다.
   * 작업 로그의 내용을 통해 직전 작업 세션의 상태와 오늘의 작업 계획을 파악합니다.
   * 작업 로그는 세션 시작 시 한 번만 읽고, 이후에는 다음 시점에 업데이트합니다:
     - 작업 하나를 완료할 때
     - 개발자가 작업 로그 업데이트를 요청할 때
     - Git에 커밋/푸시할 때
     - 업무 세션을 마무리할 때

2. **관련 작업 문서 식별**:
   * 작업 로그를 기반으로 현재 진행해야 할 작업을 식별합니다.
   * 명시적으로 작업 번호가 주어지면 해당 번호의 문서를 사용합니다.
   * "오늘 작업" 등 일반적인 지시의 경우, `../tasks/task-status.md` 파일을 참조하여 현재 진행해야 할 작업을 식별합니다.
   * 만약 `task-status.md`가 없거나 명확하지 않으면 개발자에게 확인을 요청합니다.

3. **작업 계획 문서 (`<task-number>-01-plan-*.md`) 검토**:
   * 작업의 목적, 범위, 주요 기능, 제약사항 등을 파악합니다.
   * 작업의 우선순위와 의존성을 확인합니다.
   * 작업 완료 조건과 성공 기준을 이해합니다.

4. **작업 체크리스트 (`<task-number>-02-action-checklist-*.md`) 검토**:
   * 현재까지의 진행 상황을 파악합니다.
   * 다음에 수행해야 할 작업 항목을 식별합니다.
   * 작업 항목의 우선순위와 의존성을 확인합니다.
   * 작업 체크리스트는 작업 진행 중 실시간으로 업데이트합니다.

5. **관련 아키텍처 문서 검토**:
   * 작업과 관련된 아키텍처 문서를 검토합니다.
   * 시스템 구조, 컴포넌트 간 관계, 데이터 흐름 등을 이해합니다.
   * 작업이 시스템에 미치는 영향을 파악합니다.

6. **작업 보고서 (`<task-number>-03-report-*.md`) 검토**:
   * 최근까지의 작업 진행 중 발생했던 주요 이슈, 해결 과정, 결정 사항 등을 파악합니다.
   * 작업 보고서는 작업 진행 중 실시간으로 업데이트합니다.

7. **작업 준비 완료 및 개발자에게 확인**:
   * 위 모든 문서 검토 후, 작업 시작 준비가 되었음을 개발자에게 알립니다.
   * 예: "개발자, Task #00X 관련 문서 검토 완료했습니다. [다음 작업 항목]부터 진행하겠습니다."

## 3. 작업 진행 중 원칙

1. **작업 문서 실시간 업데이트**:
   * 작업 체크리스트와 보고서는 작업 진행 중 실시간으로 업데이트합니다.
   * 코드 변경, 이슈 발생, 해결 과정 등을 즉시 기록합니다.
   * 작업 문서의 변경 사항은 Git에 커밋하기 전에 반드시 반영합니다.

2. **작업 로그 주기적 업데이트**:
   * 작업 로그는 다음 시점에 업데이트합니다:
     - 작업 하나를 완료할 때
     - 개발자가 작업 로그 업데이트를 요청할 때
     - Git에 커밋/푸시할 때
     - 업무 세션을 마무리할 때
   * 작업 로그에는 해당 시점까지의 주요 진행 상황, 이슈, 결정 사항 등을 요약하여 기록합니다.

3. **문서 및 코드 전체 읽기 후 수정/삭제 (CRITICAL)**:
    *   기존 코드나 문서를 수정하거나 삭제하기 전에는 **반드시 해당 파일의 전체 내용 또는 관련된 모듈/기능의 전체 내용을 주의 깊게 읽고 완전히 이해해야 합니다.**
    *   부분적인 이해만으로 수정/삭제를 진행하여 예기치 않은 사이드 이펙트나 정보 유실이 발생하지 않도록 합니다. 이 규칙은 모든 파일 수정/삭제 작업에 적용됩니다.

4. **체크리스트 기반 작업 수행**:
    *   식별된 작업 체크리스트의 항목에 따라 순차적으로 작업을 진행합니다.

5. **문서 일관성 유지**:
    *   작업 중 코드 변경 사항이 기존 문서(작업 계획, 아키텍처 문서 등)와 불일치를 유발할 경우, 관련 문서를 함께 업데이트하거나 개발자에게 보고하여 지침을 받습니다.

6. **질문 및 확인**:
    *   작업 지침이 불분명하거나, 예상치 못한 문제에 직면하거나, 여러 해결 방안 중 선택이 필요할 경우, 주저하지 말고 개발자에게 질문하고 명확한 지침을 받습니다.

## 4. 커뮤니케이션

*   작업 진행 상황, 주요 결정 사항, 문제 발생 시 즉시 개발자에게 보고합니다.
*   `.caretrules` 및 개발자가 정의한 페르소나와 커뮤니케이션 스타일을 준수합니다.
*   문서 작성 시 용어 및 호칭 사용은 [Project Documentation Guide](../../development/documentation-guide.md)의 가이드라인을 따릅니다.

이 가이드는 지속적으로 업데이트될 수 있습니다. 항상 최신 버전의 가이드를 숙지하고 따라주시기 바랍니다.
